# demo-durable-streams

Runnable end-to-end example using the **Durable Streams transport**. The backend appends every trace batch to a durable, resumable HTTP log; the browser reads it back over plain HTTP - no in-process broker.

This is the same fake document-processing workflow as `demo-sse` / `demo-ws`, but persisted via [`@durable-streams/server`](https://durablestreams.com) instead of an ephemeral pub/sub.

```bash
bun install         # at the repo root
bun demo:ds         # boots durable-streams server (:4437) + app server (:8789) + web (:5175)
```

Open http://localhost:5175. Click **Run successful workflow** or **Run failing workflow** and watch the cards populate live. Reload the page: history replays from offset `0`.

### What this shows

- **Server** (`src/server.ts`) - boots `DurableStreamTestServer` on port 4437 in-process (production: point at [durablestreams.com](https://durablestreams.com) or a self-hosted [Caddy-plugin](https://durablestreams.com/deployment) deployment instead). Composes `LiveTraceLayer + TraceSinkLive + DurableStreamsTransportLayer({ baseUrl })`. The transport groups events by `TraceScope` and appends NDJSON batches to `trace/<scope.type>/<scope.id>`.
- **Workflow** (`src/workflow.ts`) - identical to `demo-sse` / `demo-ws`.
- **Web** (`src/web/`) - uses `@durable-streams/client`'s `stream({ url, live: true, contentType: "application/x-ndjson" })` directly. NDJSON chunks are split per line, parsed, and dispatched into `getTraceStore()`. Vite proxies `/ds/*` → `http://127.0.0.1:4437/*` so the browser doesn't need CORS.

### Why durable streams for live traces

- **Resumable.** Save the offset, reload, continue from where you left off - no events lost.
- **Multi-pod safe.** Multiple backend workers can append to the same stream; multiple browsers can tail it.
- **CDN-friendly historical reads.** Past offsets are immutable, cacheable at the edge.
- **No infra in your app process.** The pub/sub state lives in the durable-streams server, not your in-memory broker.

See the [Durable Streams quickstart](https://durablestreams.com/quickstart) and [deployment guide](https://durablestreams.com/deployment) for production hosting options.
